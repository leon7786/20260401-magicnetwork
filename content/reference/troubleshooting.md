### [zh-Hans]

# 故障排查 / Troubleshooting

按“下载异常 → 架构异常 → 语言异常”的顺序排查，通常能快速定位问题。

## 下载按钮打不开
1. 先检查网络与代理配置。
2. 尝试直接访问按钮链接的 GitHub release 页面。
3. 若是 iOS，请优先走 App Store。

## 下载到错误架构版本
1. 确认页面按钮说明（x64 / arm64 / Intel / Apple Silicon）。
2. 某些项目命名不统一时，优先回退到 release 页面手动选择。

## 语言显示不符合预期
1. 检查浏览器首选语言顺序。
2. 强制刷新页面缓存后再试。

### [zh-Hant-TW]

# 故障排查 / Troubleshooting

建議依序排查「下載異常 → 架構異常 → 語言異常」，通常可快速定位問題。

## 下載按鈕無法打開
1. 先檢查網路與代理設定。
2. 嘗試直接訪問按鈕對應的 GitHub release 頁面。
3. 若是 iOS，請優先使用 App Store。

## 下載到錯誤架構版本
1. 確認頁面按鈕說明（x64 / arm64 / Intel / Apple Silicon）。
2. 若專案命名不一致，先回退至 release 頁面手動選擇。

## 語言顯示不符合預期
1. 檢查瀏覽器語言偏好順序。
2. 清除快取並強制重新整理。

### [en]

# Troubleshooting

Check issues in this order: download failure → architecture mismatch → language mismatch.

## Download button does not open
1. Check network and proxy availability.
2. Open the corresponding GitHub release page directly.
3. For iOS, use App Store first.

## Wrong architecture downloaded
1. Verify the button label (x64 / arm64 / Intel / Apple Silicon).
2. If asset naming is inconsistent, fallback to release page and choose manually.

## Wrong language rendered
1. Check browser preferred language order.
2. Hard-refresh page cache and retry.
